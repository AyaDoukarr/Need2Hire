IA de Qualification du Besoin de Recrutement

SQORUS · IA RH

Projet AXA · 2026

LE PROBLÈME

Le besoin manager = imprécis, subjectif, non structuré

"Je cherche un bon profil data senior autonome"

SYMPTÔMES

  • Expressions subjectives : "bon", "autonome", "senior"
  • Aucun critère objectif pour filtrer les CV
  • Incohérence avec la taxonomie RH groupe

IMPACT

  • 2-3 semaines d'échanges RH/manager
  • Mauvais candidats →二次 entretiens
  • Délai de recrutement allongé

OBJECTIF

Transformer une demande manager vague en fiche de poste AXA structurée

Objectif : permettre aux RH de lancer un recrutement avec des critères objectifs

LA SOLUTION

Une application Streamlit avec IA multi-agents

INPUT

  • Texte libre (langage naturel)
  • Audio (enregistrement vocal)
  • Fichier (fiche existante)

TRAITEMENT

  • IA multi-agents (Groq/Llama)
  • Logique RAG (taxonomie)
  • Scoring 5 dimensions

OUTPUT

  • Fiche poste AXA (10+ champs)
  • Critères évaluation
  • Questions entretien
  • Rapport qualification

PARCOURS UTILISATEUR

1Saisie
2Validation
3Analyse
4Génération
5Restitution

DÉTAIL DES ÉTAPES

  • 1. Saisie : texte, audio, fichier
  • 2. Validation : longueur min, nettoyage
  • 3. Analyse : famille (45) + sous-famille
  • 4. Génération : fiche JSON structurée
  • 5. Restitution : fiche + score + brief

RÉFÉRENTIEL MÉTIER (RAG)

SOURCES DE DONNÉES

  • axa_taxonomie_metiers.csv : 45 familles + sous-familles
  • axa_referentiel_metiers.csv : compétences par famille
  • axa_fiches_poste_it_enrichi.csv : postes IT détaillés
  • familles_metiers_axa.json : mots-clés structurés

FONCTIONNEMENT RAG

  • Chargement des référentiels au démarrage
  • Extraction mots-clés selon famille détectée
  • Injection dans le prompt LLM comme contexte
  • Alignement automatique réponse/standard AXA
RAG = Retrieval-Augmented Generation. Le LLM génère avec le contexte des référentiels AXA.

POURQUOI PLUSIEURS AGENTS ?

  • Séparation responsabilités : chaque agent expert dans son domaine
  • Confidence routing : Python si certain, LLM si incertain
  • Quality gates : validation à chaque étape
  • Maintenabilité : agent modifiable sans Impacter les autres

LOGIQUE DE SCORING

5 DIMENSIONS (20 pts chacune)

  • 1. Complétude : champs structurants présents
  • 2. Précision : termes flous, info manquantes
  • 3. Screening : critères exploitables
  • 4. Missions : clarté, actionnabilité
  • 5. Conformité : structure AXA

INTERPRÉTATION

  • 0-39 : À clarifier avant usage
  • 40-69 : Utilisable avec ajustements
  • 70-84 : Opérationnel
  • 85-100 : Excellence

Confiance : 25-95% (basé sur cohérence)

Objectif : fournirdescription objectif pour aider la décision RH
