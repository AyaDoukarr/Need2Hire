Need2Hire

IA de Qualification du Besoin de Recrutement

SQORUS | IA RH

Projet AXA - 2026

Contexte Métier

Les demandes de recrutement des managers sont souvent imprécises

  • Expressions subjectives : "bon profil", "autonome", "senior"
  • Manque de critères objectifs
  • Informations manquantes : skills, expérience, contexte

Impact : 2-3 semaines d'échanges RH/manager

Problème Adressé

  • Inefficacité
    Aller-retour multiples
  • Incohérence
    Pas de standard RH
  • Subjectivité
    Critères flous
  • Perte de temps
    Screening difficile

Objectif

Transformer une demande vague en fiche de poste structurée et exploitable

Solution Proposée

Entrée

  • Texte libre
  • Audio (voix)
  • Fichier existant

Traitement

  • IA multi-agents
  • Logique métier (RAG)
  • Taxonomie AXA

Sortie

  • Fiche AXA structurée
  • Score de qualité
  • Critères screening

Parcours Utilisateur

  1. Saisie : Besoin en texte ou audio
  2. Validation : Vérification minimale
  3. Analyse : Détection famille/sous-famille
  4. Extraction : Génération fiche JSON
  5. Scoring : 5 dimensions (0-100)
  6. Restitution : Fiche + brief + rapport

Architecture Fonctionnelle

  • Data Layer = Référentiels métier AXA (45 familles, mots-clés, compétences)

Flux de Données

RAG : La taxonomie AXA fournit le contexte pour guider le LLM

Agents IA Spécialisés

  • Family Router : Détection famille métier (45 familles)
  • Subfamily Router : Détection sous-famille
  • Extractor : Génération fiche structurée JSON
  • Quality : Validation structure et contenu
  • Market Benchmark : Analyse concurrentielle

Stratégie hybride : Matching Python (rapide) → LLM si incertain

Référentiels Métier (Logique RAG)

Data Layer

  • axa_taxonomie_metiers.csv
  • axa_referentiel_metiers.csv
  • axa_fiches_poste_it_enrichi.csv
  • familles_metiers_axa.json

Usage

  • 45 familles métier AXA
  • Mots-clés par famille
  • Sous-familles
  • Compétences techniques
  • Exemples de postes

Le contexte métier est injecté dans le prompt LLM pour des réponses alignées

Scoring et Aide à la Décision

5 Dimensions

  1. Complétude (20 pts)
  2. Précision (20 pts)
  3. Screening (20 pts)
  4. Missions (20 pts)
  5. Conformité AXA (20 pts)

Score Global

  • 0-39 : À clarifier
  • 40-69 : À ajuster
  • 70-84 : Opérationnel
  • 85-100 : Excellence

Confiance : 25-95%

Points Forts

  • Architecture modulaire : Séparation des responsabilités
  • Hybridation : Python (rapide) + LLM (intelligent)
  • Standard AXA : Taxonomie et format métier
  • Aide décision : Score objectif + confiance
  • Maintenance : Agents indépendants

Limites Actuelles

  • Dépendance LLM : Groq/Llama externe
  • Rate limits : Limite API
  • Pas d'intégration ATS : Entrée manuelle
  • Qualité entrée : Garbage in = garbage out

Perspectives

Court Terme

  • Intégration Workday
  • Multi-langues

Moyen Terme

  • Benchmark salarial
  • Matching candidats

Long Terme

  • Prédiction succès
  • Automatisation complète
